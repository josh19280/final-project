方法 0
實作進度：已完成
製作日期：2024/12/20
實作最基本的 BMP 讀取與寫回流程。
Encoder 將 BMP 圖檔拆分為 R / G / B 三個 channel，並以 ASCII 格式儲存，同時輸出影像尺寸。
Decoder 讀取 R / G / B 與尺寸資訊後，重建 BMP 圖檔，並以 diff / cmp 驗證重建影像與原始影像完全一致，用以確認 BMP 解析與資料順序正確。

方法 1（DCT、量化與重建驗證）
實作進度：已完成
製作日期：2024/12/26
方法 1 主要用來驗證 JPEG-like 壓縮流程是否正確，包含：
RGB → YCbCr 色彩空間轉換
以 8×8 block 為單位的 2D-DCT
頻域係數量化與量化誤差計算
Encoder 會輸出量化表、量化後係數（qF）與量化誤差（eF），並列印頻域 SQNR。
Decoder 分為兩種模式：
Decoder 1(a)：僅使用量化後係數進行重建，影像與原圖存在誤差
Decoder 1(b)：加入量化誤差進行重建，可完整還原原始影像，並以 cmp 驗證結果正確

方法 2（進階編碼與完整解碼流程）
實作進度：Encoder 與 Decoder 2(a)、2(b) 皆已完成
製作日期：2024/12/28
方法 2 延續前一階段的系統架構，將整體影像編碼與解碼流程整合為更完整的實作。
Encoder 可正確產生對應之頻域與量化資料，Decoder 則支援：
Decoder 2(a)：使用量化後頻域資料進行影像重建
Decoder 2(b)：加入量化誤差進行完整解碼，重建結果與原始影像一致
